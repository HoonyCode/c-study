### 형변환
- C 언어에서는 각 변수들에는 고유의 형(Type)이 있다
- 가끔씩 프로그래밍을 하다 보면 형이 다른 변수 끼리 대입을 하는 연산이 필요하다 
- 예시) 형이 다른 변수 끼리 댕ㅂ을 하는 연산이 필요할 떄
    ```C
    double sum;
    int A = 10;
    float B = 20.0;
    sum = A + B;
    ```
- 실행 결과 : 30
- 컴파일 오류 경고가 나오게 된다.

> 형 변환

- 우리는 경고가 나오지 않게 대입을 하는 방법은 없을까?
  - 서로의 형을 맞추어 주자

```C
#include <stdio.h>
int main(){
    int a;
    double b;
    b = 2.4;
    a = (int)b;
    printf("%d", a);
}
// 출력 결과 : 2
```
- 경고 메세지가 안 나오는 이유는 강제로 형변환(캐스팅)을 했기 때문이다.

```C
(바꾸려는 형) 변수 이름
```

> 암시적 형 변환과 형 축소

![image](https://user-images.githubusercontent.com/44612896/135736759-ed2e7ce9-6b18-4c82-a313-7311cbf8c8da.png)

- 암식적 형 변환
  - 자료형의 크기가 큰 쪽, 표현 범위가 넓은 쪽으로 자동 변환되는 것을 형 확장이라고 한다.

- 형 축소
  - 자료형 크기가 작은 쪽, 표현 범위가 좁을 쪽으로 변환되는 것이 형 축소이다.
  - 형 축소에서 컴파일 경고가 나오지 않도록 만드는 것을 형 변환(type casting)이라고 한다.
  - 강제로 자료형을 변환한다고 해서 명시적 형 변환이라 부르기도 한다.
    

### 포인터
- 메모리 상에 위치한 특정한 데이터의 (시작)주소값을 보관하는 변수를 말한다.


- 표현방식
```C
(포인터에 주소값이 저장되는 데이터 형) *(포인터 이름)
(포인터에 주소값이 저장되는 데이터 형)* (포인터 이름)
```
> &연산자

- &연산자는 피연산자의 주소값을 부르는 연산자이다.

- 사용방법
```C
int a = 10;
int* p = &a;
/// p는 a변수의 주소값을 가르키고 있다.
```

> 포인터가 타입을 가지는 이유
- 어차피 주소값은 32 비트 시스템에서 항상 4바이트이고, 64 비트 시틑넴에서는 8바이트 인데 그냥 pointer라는 타입을 만들어버리면 안 될까?
```C
int a; 
pointer *p; 
p = &a;
*p = 4;
```
- 위의 코드를 생각해 보자 메로이에 a를 위해서 4바이트 짜리 공간을 마련해줬고, 마찬가지로 p를 위해 메모리 상에 8 바이트 짜리 공간을 마련하였습니다. 그리고 p에 a의 주소값을 잘 전달하였지요
- 문제는 아래 코드입니다.
```C
*p = 4;
```
- 포인터 p에는 명백히 변수 a의 주소값들이 들어 있습니다. 여기서 문제는 a가 메모리에서 차지하는 모든 주소들의 위치가 들어 있는 것이 아니라 `시작 주소`만 들어가 있다는 점이다. 따라서 `*p` 라고 했을 떄 컴퓨터는 메모리에서 얼마만큼을 읽어야 할지 알 길이 없다.
```C
int a; 
int *p; 
p = &a; 
*p = 4;
```
- 위 코드처럼 명시한다면 포인터 p가 int*라는 사실을 보고 이 포인터는 int 데이터를 가리키는 구나! 라고 알게 되어 시작 주소로 부터 정확히 4바이트를 읽어 들어 값을 바꾸게 된다

즉, 포인터도 변수라고 생각하면 편하다❗️


### 함수

> 함수(function)란?
- 특정한 값을 입력 받아, 이 값을 가지고 어떤 수식을 통해 결과를 내보는 것을 말한다.

```
y = f(x)
```

> 함수 선언 방법

```C
int function(int a, int b){
  return a+b;
}
```
```C
리턴타입 함수이름(매개변수){

  //수식

  return 리턴타입
}
```

> 함수의 장점
- 한번 선언해 놓은 함수는 계속 재사용 할 수 있다.
- 코드의 중복을 줄일 수 있다.
